# Life Log Stable

一个本地优先的个人生命力日志原型。它不需要登录、数据库、云同步或 AI API，记录保存在当前设备的当前浏览器中，并支持 JSON 与 Markdown 导出。

## 使用方式

### 本地打开

直接打开 `index.html` 即可查看界面。  
如果要测试离线缓存和添加到 iPhone 主屏幕，建议用任意静态服务器或部署到一个固定网址。

### 部署

把整个 `life-log-stable` 文件夹部署为静态站点即可。入口文件是：

```text
index.html
```

请尽量保持网址稳定。手机浏览器的本地数据通常和网址绑定；如果换了网址，旧记录不会自动跟过去，需要先导出 JSON 再导入。

## 数据保存

- 输入时会自动保存。
- 点击“保存今日记录”会进行一次明确保存。
- “今日记录用时”会在开始填写后自动累计。
- 离开页面、切到后台、切换页面或连续 2 分钟无操作时，会暂停本次计时。
- 一天打开多次时，用时会自动加总。

## 备份与导出

在“导出”页面可以：

- 导出 JSON 备份：用于未来恢复数据。
- 导出 Markdown：用于阅读、归档，或发给 Codex/GPT 做每周复盘。
- 导入 JSON 备份：恢复或迁移记录。

建议每周至少导出一次 JSON 备份。

## iPhone 连续使用建议

- 用 Safari 打开固定网址后，可以选择“添加到主屏幕”。
- 尽量固定同一个部署网址；浏览器本地记录通常和网址绑定。
- 如果需要换网址、换浏览器、重新部署到新站点，请先导出 JSON 备份。
- 每次输入都会自动保存；切到后台或关闭页面前，也会尝试立即保存当前表单。
- 一周导出一次 JSON，是目前这个本地优先原型最稳妥的备份方式。

## 当前 MVP 能力

- 每日生命力日志
- 热启动 Priming 页面
- 口说转写粘贴
- 今日卡片生成
- 热启动写入今日备忘
- 全字段中文提示语
- 多选观察标签
- 自动累计记录用时
- 自动保存与手动保存
- 今日目录与快速定位
- 历史记录按日期查看
- 最近 7 天周复盘
- 庶务记录
- 运动分钟与散步/走路公里数记录
- JSON 备份导出/导入
- Markdown 导出
- PWA 离线缓存基础支持

## 当前字段结构

```text
备忘
灵修
睡眠
身体
庶务
运动
清醒时间
内在
能量
创造
关系
自由
```

旧版本里的“饮食”和“生活照料”内容会在新版本中合并到“庶务”。旧版本里的“放松与娱乐”会合并到“清醒时间”。旧版本里的“灵感”会合并到“创造”。旧版本里的“梦境”会在导出和回看时并入“自由”。

## 热启动流程

```text
打开热启动
粘贴口说转写
生成今日卡片
必要时手动调整卡片
写入今日备忘
全天继续补 Life Log
```

热启动可以一天多次使用：早起可以用一次，面对重要困难任务时也可以再用一次。

热启动卡片包括：

```text
今日主线
第一动作
过程画面
阻力预案
生活托底
今日产出
```
